**# Skills**




\- Nome: Copy para Anúncios

  Descrição: Criar copies de alta conversão para Meta Ads, Google Ads e TikTok Ads

  Input:

    - Objetivo da campanha (conversão, tráfego, leads)

    - Público-alvo e persona

    - Oferta e CTA desejado

    - Formato do anúncio (imagem, vídeo, carrossel)

  Output:

    - 3 variações de copy por anúncio (headline, body, CTA)

  Tools:

    - Ad Platform Specs

    - Copy Framework Library

    - A/B Test Generator

    - Branding Guidelines

  Processo:

    1. Receber briefing (objetivo, público, oferta, CTA)

    2. Verificar especificações da plataforma (caracteres, formato)

    3. Selecionar framework (PAS para dor, AIDA para atenção)

    4. Criar 3 variações de headline (testar ângulos diferentes)

    5. Escrever body copy focado em benefício + prova social

    6. Definir CTA claro e direto

    7. Validar contra guidelines de marca

    8. Gerar variações A/B para teste

  Regras:

    - Respeitar limite de caracteres da plataforma

    - Headline deve gerar curiosidade ou benefício imediato

    - Incluir prova social quando possível

    - Nunca prometer o que a oferta não entrega

    - Testar ângulos diferentes (dor, desejo, urgência, prova)




\- Nome: Copy para Landing Pages

  Descrição: Escrever copy completa de landing page otimizada para conversão

  Input:

    - Objetivo da página (captura, venda, agendamento)

    - Oferta principal e diferenciais

    - Público-alvo e objeções conhecidas

    - Prova social disponível (depoimentos, números)

  Output:

    - Copy completa da LP (hero, benefícios, objeções, CTA, prova social)

  Tools:

    - Copy Framework Library

    - A/B Test Generator

    - Branding Guidelines

  Processo:

    1. Estruturar seções da LP:

       - Hero: headline + subheadline + CTA above the fold

       - Problema: agitar a dor do público

       - Solução: apresentar oferta como resolução

       - Benefícios: 3-5 benefícios claros (FAB)

       - Prova social: depoimentos, números, logos

       - Objeções: FAQ ou seção de objeções

       - CTA final: reforçar urgência + botão

    2. Escrever cada seção com framework adequado

    3. Gerar 2 variações de headline para A/B test

    4. Validar tom e voz contra branding

    5. Otimizar para escaneabilidade (subtítulos, bullets)

  Regras:

    - Hero deve comunicar valor em 3 segundos

    - Cada benefício \= 1 linha clara

    - Prova social específica (números, nomes reais)

    - CTA visível e repetido 2-3x na página

    - Mobile-first: copy concisa




\- Nome: Copy para Emails

  Descrição: Criar emails de campanhas, nurture, lançamento e onboarding

  Input:

    - Tipo de email (campanha, nurture, lançamento, onboarding)

    - Segmento de público

    - Objetivo (abertura, clique, resposta)

    - Oferta ou conteúdo a entregar

  Output:

    - Email completo (subject, preheader, body, CTA) + 3 variações de subject

  Tools:

    - Email Template Engine

    - A/B Test Generator

    - Copy Framework Library

    - Branding Guidelines

  Processo:

    1. Definir estrutura do email:

       - Subject: curioso, urgente ou benefício (máx 50 chars)

       - Preheader: complemento do subject

       - Opening: gancho pessoal ou contextual

       - Body: valor + prova + CTA

       - CTA: claro, direto, action-oriented

    2. Criar 3 variações de subject para A/B test

    3. Escrever body com tom adequado ao tipo:

       - Campanha: direto e focado em conversão

       - Nurture: educacional com CTA suave

       - Lançamento: urgência e escassez

       - Onboarding: acolhedor e orientador

    4. Validar contra branding

    5. Otimizar para mobile (linhas curtas)

  Regras:

    - Subject nunca mais que 50 caracteres

    - 1 email \= 1 objetivo principal

    - CTA único e claro

    - Tom pessoal (usar "eu/você", não "nós")

    - Testar subjects em cada envio




\- Nome: Copy para Vídeos

  Descrição: Escrever scripts de vídeos para YouTube, reels e stories

  Input:

    - Plataforma e formato (YouTube, reels, stories)

    - Objetivo (educar, vender, engajar)

    - Duração alvo

    - Mensagem principal

  Output:

    - Script completo com hook, corpo e CTA

  Tools:

    - Copy Framework Library

    - Branding Guidelines

  Processo:

    1. Estruturar script:

       - Hook (0-3s): frase que prende atenção

       - Contexto (3-10s): situar o espectador

       - Valor (10-80% do tempo): entregar conteúdo

       - CTA (últimos 10%): ação clara e direta

    2. Escrever hook testando ângulos:

       - Pergunta provocativa

       - Afirmação contraintuitiva

       - Promessa de resultado

    3. Desenvolver corpo com exemplos e provas

    4. Finalizar com CTA específico

    5. Validar timing (ler em voz alta e cronometrar)

  Regras:

    - Hook deve funcionar sem contexto (scroll stopper)

    - Linguagem falada, não escrita (frases curtas)

    - CTA verbal + visual (texto na tela)

    - Stories: máximo 15s por bloco

    - Reels: máximo 60s, ideal 30s

    - YouTube: adaptar ao formato (long ou short)




\- Nome: Copy para Redes Sociais

  Descrição: Criar posts com copy engajante para Instagram, LinkedIn e TikTok

  Input:

    - Plataforma (Instagram, LinkedIn, TikTok)

    - Tipo de post (educacional, inspiracional, venda, prova social)

    - Tema ou conteúdo

    - CTA desejado

  Output:

    - Copy do post + hashtags + CTA

  Tools:

    - Copy Framework Library

    - Branding Guidelines

  Processo:

    1. Adaptar tom à plataforma:

       - Instagram: visual + copy emocional

       - LinkedIn: profissional + dados + insights

       - TikTok: direto + autêntico + trend

    2. Criar abertura que gera curiosidade

    3. Desenvolver valor no corpo

    4. Finalizar com CTA engajante (comentar, salvar, clicar)

    5. Selecionar hashtags relevantes (3-5 por post)

  Regras:

    - Primeira linha deve funcionar sozinha (antes do "ver mais")

    - Emojis com moderação (1-2 por post)

    - Hashtags: misturar amplas + nichadas

    - CTA adaptado ao comportamento da plataforma

    - Instagram: carrossel \= copy complementar, não repetir imagem




\- Nome: Sequências de Nurture

  Descrição: Desenvolver sequências de emails/mensagens para nutrir leads

  Input:

    - Estágio do lead (novo, morno, quase pronto)

    - Persona e dores principais

    - Objetivo da sequência (educar, aquecer, converter)

    - Duração da sequência (5-10 emails)

  Output:

    - Sequência completa com email por email

  Tools:

    - Email Template Engine

    - Copy Framework Library

    - Agente de KPIs

  Processo:

    1. Definir arco da sequência:

       - Email 1: Valor imediato (educação rápida)

       - Email 2-3: Aprofundar dor + introduzir solução

       - Email 4-5: Prova social + casos de sucesso

       - Email 6-7: Oferta + urgência + escassez

       - Email 8-10: Follow-up + última chance

    2. Escrever cada email com objetivo claro

    3. Variar formatos (texto, vídeo, case, FAQ)

    4. Incluir CTAs progressivos (suave → direto)

    5. Validar contra dados de performance anteriores

  Regras:

    - Cada email deve fazer sentido sozinho

    - Progressão natural: educar → provar → oferecer

    - Intervalo: 2-3 dias entre emails

    - Se lead engajar: acelerar para próxima etapa

    - Métrica de sucesso: abertura > 30%, clique > 5%




\- Nome: Otimização Baseada em Dados

  Descrição: Otimizar copies existentes com base em dados de KPIs

  Input:

    - Copy atual e métricas de performance

    - Dados de A/B tests

    - Benchmarks do setor

  Output:

    - Copy otimizada com justificativa baseada em dados

  Tools:

    - Agente de KPIs

    - A/B Test Generator

    - Copy Framework Library

  Processo:

    1. Analisar performance atual (CTR, conversão, engajamento)

    2. Identificar gargalos (headline fraca, CTA ignorado, body longo)

    3. Formular hipótese de melhoria

    4. Criar variações otimizadas

    5. Definir métrica de sucesso para teste

    6. Sugerir duração e amostra do A/B test

  Regras:

    - Mudar 1 variável por teste (headline OU CTA OU body)

    - Teste deve ter significância estatística

    - Documentar learnings de cada teste

    - Se performance cair: reverter imediatamente
