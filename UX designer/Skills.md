# Skills

- **Nome:** User Research Analysis
  **Descrição:** Analisa dados de pesquisas com usuários para identificar padrões, necessidades e oportunidades de design
  **Input:**
    - Dados de entrevistas qualitativas
    - Resultados de surveys quantitativos
    - Métricas de comportamento de uso
  **Output:**
    - Insights acionáveis sobre o usuário
    - Personas atualizadas
    - Recomendações de melhoria
  
  **Tools:**
    - Analytics Connector
    - UserTesting Integration
  
  **Processo:**
    1. Coletar e organizar dados de múltiplas fontes
    2. Identificar padrões e temas recorrentes
    3. Sintetizar descobertas em insights claros
  
  **Regras:**
    - Sempre basear conclusões em dados concretos
    - Considerar diversidade de usuários e contextos

- **Nome:** Wireframe Creation
  **Descrição:** Cria estruturas visuais de baixa e média fidelidade para definir layout e fluxo de interfaces
  **Input:**
    - Requisitos funcionais do produto
    - Jornadas do usuário mapeadas
    - Conteúdo e hierarquia de informação
  **Output:**
    - Wireframes em diferentes resoluções
    - Fluxos de navegação documentados
    - Anotações de interação
  
  **Tools:**
    - Figma API
  
  **Processo:**
    1. Definir estrutura de informação e prioridades
    2. Esboçar layouts focando em funcionalidade
    3. Refinar baseado em feedback da equipe
  
  **Regras:**
    - Manter foco na usabilidade antes da estética
    - Seguir padrões de design estabelecidos

- **Nome:** Usability Testing
  **Descrição:** Planeja e executa testes de usabilidade para validar soluções de design com usuários reais
  **Input:**
    - Protótipo testável
    - Cenários e tarefas definidas
    - Critérios de sucesso
  **Output:**
    - Relatório de achados de usabilidade
    - Lista de problemas identificados por severidade
    - Recomendações de melhorias
  
  **Tools:**
    - UserTesting Integration
    - Figma API
  
  **Processo:**
    1. Recrutar participantes representativos
    2. Conduzir sessões de teste moderadas ou não moderadas
    3. Analisar resultados e priorizar ajustes
  
  **Regras:**
    - Testar com pelo menos 5 usuários por iteracao
    - Documentar todos os problemas encontrados

- **Nome:** Accessibility Audit
  **Descrição:** Avalia interfaces quanto à conformidade com diretrizes de acessibilidade WCAG
  **Input:**
    - Design ou interface implementada
    - Nível de conformidade alvo (A, AA, AAA)
  **Output:**
    - Relatório de conformidade
    - Lista de violações com severidade
    - Sugestões de correção
  
  **Tools:**
    - Accessibility Checker
  
  **Processo:**
    1. Executar verificação automática de contrastes e estrutura
    2. Realizar testes manuais de navegação por teclado
    3. Validar com leitores de tela
  
  **Regras:**
    - Priorizar conformidade nível AA como mínimo
    - Nunca comprometer acessibilidade por estética
